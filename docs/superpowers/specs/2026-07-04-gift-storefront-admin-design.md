# AI 送礼商城与管理后台设计

日期：2026-07-04  
状态：已确认，待实施计划

## 1. 目标

在现有 FastAPI 后端基础上，建设两套可实际运行的界面：

- 面向顾客的响应式 AI 送礼商城；
- 面向管理员的商品管理后台。

第一版解决“发现礼物、获得 AI 推荐、查看商品、收藏对比、分享和联系购买”的完整流程。第一版不包含购物车、订单、在线支付或物流。

## 2. 视觉方向

选定方向为“贴心对话”。视觉目标以用户提供的 `index(1).html` 原型为基础，并以已确认的第三套视觉方案为实现基准。

- 主色为珊瑚红到暖橙色渐变；
- 使用温暖、精致、友好的礼品顾问语气；
- 使用清晰的中文字体、圆角界面和充足留白；
- 手机端使用底部导航，电脑端扩展为宽屏双栏布局；
- 使用统一的专业图标库，不使用表情符号代替正式图标；
- 商品区域使用真实商品图片，不使用占位图或 CSS 模拟图形。

## 3. 用户端范围

### 3.1 首页与 AI 咨询

- 首页以 AI 顾问对话为第一视觉重点；
- 用户依次选择送礼对象、场景、预算和偏好；
- 显示清楚的步骤状态，并允许返回修改已选条件；
- AI 失败时允许重试，并提供进入普通商品浏览的降级入口；
- AI 结果显示商品、推荐理由、适用对象和预算匹配信息。

### 3.2 商品浏览

- 支持商品列表、分类和品牌筛选；
- 支持商品详情、封面、多图、价格、说明和推荐标签；
- 支持收藏商品；
- 支持将多个收藏商品加入对比清单；
- 不展示购物车、下单或支付入口。

### 3.3 联系购买

AI 推荐结果和商品详情都提供以下入口：

- 微信咨询：展示后台配置的微信号和二维码；
- 电话咨询：移动端使用 `tel:` 一键拨号，桌面端显示并允许复制号码。

联系方式由后台维护，前端不得硬编码业务联系方式。

### 3.4 分享

- AI 推荐结果支持分享整套送礼方案；
- 商品详情支持分享单个商品；
- 支持 Web Share API 的移动设备优先调用系统分享；
- 不支持 Web Share API 时提供复制链接；
- 桌面端额外提供分享二维码；
- 分享内容包含标题、封面、推荐理由和可访问的详情链接；
- 分享文案和默认封面可在后台配置。

### 3.5 导航与响应式布局

手机端底部导航为：首页、咨询、清单、我的。电脑端保持相同信息架构，使用顶部或侧边导航与双栏内容布局。支持常见手机宽度以及 1280px 以上桌面宽度，不出现横向溢出、遮挡或不可点击控件。

## 4. 管理后台范围

### 4.1 登录与权限

- 使用用户名和密码登录；
- 普通用户不得访问后台页面或后台写接口；
- 后台路由和后端 API 同时校验管理员身份；
- 登录过期后清理本地状态并返回登录页；
- 不在前端代码、日志或错误提示中暴露凭据和令牌。

### 4.2 仪表盘

展示商品数量、上架商品数量、分类数量、品牌数量和最近更新商品。第一版不虚构销售额、订单或支付数据。

### 4.3 内容管理

- 商品：列表、搜索、新增、编辑、上下架、排序和删除；
- 图片：封面和多图上传，显示上传进度及失败原因；
- 分类：新增、编辑、排序和删除保护；
- 品牌：新增、编辑、排序和删除保护；
- 联系方式：微信号、微信二维码和联系电话；
- 分享设置：分享标题、默认文案和默认封面。

删除操作必须二次确认。存在关联数据时，分类或品牌不得被直接删除，并给出可操作的提示。

### 4.4 后台响应式要求

桌面端采用左侧导航和主内容区。手机端支持登录、查看、创建和编辑等基础管理操作；复杂表格转换为卡片或可控横向区域，主要操作保持可见。

## 5. 技术架构

### 5.1 前端

- `frontend/`：Vue 3 + Vite 用户商城；
- `admin/`：Vue 3 + Vite 管理后台；
- 使用 Vue Router 管理页面导航；
- 使用共享的 API 请求约定、数据类型和视觉变量；
- 两套应用保持独立入口、独立构建和独立部署，避免权限边界混杂。

### 5.2 后端

继续使用现有 FastAPI 应用和 SQLAlchemy 数据访问层，并补充：

- 明确的管理员权限依赖；
- 联系方式与分享设置的数据模型及接口；
- 面向前端的公开配置读取接口；
- 分享链接所需的稳定商品与推荐方案标识；
- 与现有统一响应格式一致的错误响应。

注册接口不得自动创建管理员。管理员账号应通过受控初始化流程创建或由已授权管理员提升。

### 5.3 部署

Nginx 提供用户商城、管理后台和 `/api` 反向代理。建议路由：

- `/`：用户商城；
- `/admin/`：管理后台；
- `/api/`：FastAPI；
- `/uploads/`：受控上传文件。

生产目标、域名、服务器和凭据不在本设计中推断，部署前必须由用户明确确认。

## 6. 数据流

1. 用户在商城选择送礼条件；
2. 商城调用现有 AI 推荐接口；
3. 后端返回匹配商品及推荐理由；
4. 用户进入详情、收藏、对比、分享或联系购买；
5. 管理员在后台维护商品和公开联系方式；
6. 商城通过公开只读接口获取最新配置；
7. 所有写操作经身份验证，后台写操作额外校验管理员权限。

收藏和未登录状态优先保存在浏览器本地；如果后续需要跨设备同步，再单独设计用户账户数据，不在第一版暗中扩大范围。

## 7. 错误与边界处理

- 网络失败：保留当前输入，提供重试；
- AI 服务失败：给出友好提示并允许普通浏览；
- 登录失效：清理令牌并返回登录页；
- 权限不足：显示无权访问，不展示后台内容；
- 上传失败：区分文件过大、格式不支持和网络问题；
- 保存冲突或校验失败：在对应字段附近显示原因；
- 联系方式未配置：隐藏对应入口，不显示空二维码或空号码；
- 分享能力不可用：回退到复制链接。

## 8. 测试与验收

### 8.1 自动化测试

- 后端：管理员权限、公开配置、商品写操作、上传限制和错误响应；
- 用户端：AI 条件流程、推荐结果、商品详情、收藏、对比、联系购买和分享回退；
- 后台：登录保护、商品 CRUD、图片上传、分类品牌关联保护和配置保存；
- 构建：两套 Vue 应用均能通过生产构建。

### 8.2 浏览器验收

使用 Google Chrome 验收：

- 390×844 手机视口；
- 768px 平板宽度；
- 1440×1024 桌面视口。

验收时将选定视觉图与实际页面使用相同视口并排比较，检查字体、间距、颜色、圆角、图片裁切、导航和交互状态。关键流程必须在真实后端接口上完成一次端到端验证。

## 9. 完成标准

- 用户可从首页完成 AI 送礼咨询并进入真实商品详情；
- 用户可收藏、对比、分享并通过微信或电话联系购买；
- 管理员可安全登录并维护商品、图片、分类、品牌和公开配置；
- 普通用户无法调用后台写接口；
- 手机、平板和桌面布局均可用；
- 两套前端生产构建与后端测试通过；
- 页面视觉与选定的“贴心对话”方向一致。
